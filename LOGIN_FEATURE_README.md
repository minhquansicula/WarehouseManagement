# Warehouse Management System - Login Feature

## Quick Start

### 1. Thiết lập Database

#### Option A: Oracle Database (Khuyến nghị)

**Tải JDBC Driver:**

- Tải `ojdbc11.jar` từ: https://repo1.maven.org/maven2/com/oracle/database/jdbc/ojdbc11/23.3.0.23.09/ojdbc11-23.3.0.23.09.jar
- Đặt vào: `WebContent/WEB-INF/lib/ojdbc11.jar`

**Cấu hình kết nối:**

1. Mở: `src/java/util/DBConnectionManager.java`
2. Sửa URL, USER, PASS theo database của bạn:

```java
private static final String URL = "jdbc:oracle:thin:@localhost:1521:XE";
private static final String USER = "system";
private static final String PASS = "12345";
```

**Tạo bảng users:**
Chạy SQL từ file `DATABASE_SETUP.sql`:

```sql
CREATE TABLE users (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    username VARCHAR2(100) UNIQUE NOT NULL,
    password VARCHAR2(255) NOT NULL,
    role VARCHAR2(50) DEFAULT 'USER'
);

INSERT INTO users (username, password, role) VALUES ('admin', 'admin', 'ADMIN');
INSERT INTO users (username, password, role) VALUES ('user1', 'user123', 'USER');
COMMIT;
```

#### Option B: MySQL / H2 (Xem chi tiết trong JDBC_SETUP_GUIDE.md)

### 2. Build Project

```bash
cd E:\PRACTICE\WarehouseManagment
ant clean compile war
```

### 3. Deploy WAR

1. **Nếu dùng Tomcat:**
   - Copy `dist/WarehouseManagement.war` vào `$CATALINA_HOME/webapps/`
   - Hoặc dùng Tomcat Manager upload

2. **Nếu dùng VS Code Tomcat extension:**
   - F1 → "Tomcat: Deploy or Redeploy artifact" → chọn WAR

### 4. Test Login

1. Truy cập: http://localhost:8080/WarehouseManagement/jsp/login.jsp
2. Đăng nhập:
   - Username: `admin`
   - Password: `admin`
3. Kỳ vọng: Chuyển hướng tới dashboard

## Workflow Đăng Nhập

### Flow 1: Trước Đăng Nhập

```
User → http://localhost:8080/.../jsp/dashboard.jsp
  ↓
AuthFilter: Kiểm tra session.currentUser (null)
  ↓
Redirect → /jsp/login.jsp
  ↓
Hiển thị form đăng nhập
```

### Flow 2: Đang Đăng Nhập

```
User: Form POST /login.do
  ↓
Struts ActionServlet → LoginAction.execute()
  ↓
LoginAction:
  1. Lấy username, password từ LoginForm
  2. UserDAO.authenticate(username, password)
  3. Query database: SELECT * FROM users WHERE username=? AND password=?
  4. Nếu tìm được:
     - Lưu session: currentUser, currentUserRole, currentUserId
     - Return forward("success") → /jsp/dashboard.jsp
  5. Nếu không tìm:
     - Set request attribute: loginError
     - Return forward("input") → /jsp/login.jsp
```

### Flow 3: Sau Đăng Nhập

```
User → http://localhost:8080/.../jsp/dashboard.jsp
  ↓
AuthFilter: Kiểm tra session.currentUser (có giá trị)
  ↓
chain.doFilter() → JSP được load
  ↓
Header.jsp: Hiển thị "Welcome, {currentUser}!" + Logout link
  ↓
Dashboard content hiển thị bình thường
```

### Flow 4: Đăng Xuất

```
User: Click "Logout" link (/logout.do)
  ↓
LogoutAction.execute()
  ↓
1. request.getSession().invalidate()
  2. Redirect → /jsp/login.jsp
  ↓
Session bị xóa, quay về login page
```

## Troubleshooting

### CSS/JS không tải (404)

**Vấn đề:** `GET http://localhost:8080/css/custom.css 404`
**Giải pháp:**

- ✅ Đã sửa: dùng `<%=request.getContextPath()%>/css/custom.css`
- Rebuild và redeploy WAR

### JDBC Driver Not Found

**Vấn đề:** `ClassNotFoundException: oracle.jdbc.driver.OracleDriver`
**Giải pháp:**

- [ ] Tải ojdbc11.jar từ Maven Central
- [ ] Copy vào `WebContent/WEB-INF/lib/ojdbc11.jar`
- [ ] Rebuild: `ant clean compile war`
- [ ] Redeploy WAR

### Login thất bại (Invalid username or password)

**Vấn đề:** Nhập username/password đúng nhưng vẫn lỗi
**Giải pháp:**

1. Kiểm tra database connection:
   - DBConnectionManager URL, USER, PASS đúng không?
   - Database server có chạy không?
2. Kiểm tra users table:
   - Table đã được tạo không?
   - `SELECT * FROM users;` có dữ liệu không?
3. Kiểm tra Tomcat logs:
   - `$CATALINA_HOME/logs/catalina.out`
   - Tìm exception stack trace

### Session không được lưu

**Vấn đề:** Đăng nhập thành công nhưng quay về login page
**Giải pháp:**

- Kiểm tra browser: Cookies có được lưu không?
- Kiểm tra Tomcat: Session được tạo không?
- Enable debug logging trong Tomcat
